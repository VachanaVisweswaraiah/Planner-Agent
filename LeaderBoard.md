# Calendar Scheduling Leaderboard

This project evaluates and compares multiple Small Language Models (SLMs) on a **calendar scheduling task** using a professor-provided evaluation script.  
The goal is to **identify which SLM performs best** and clearly justify the choice using a ranked leaderboard.

---

## üìå What This Does

1. **Takes model output files**  
   Each JSON file contains predictions generated by one SLM for the calendar scheduling dataset.

2. **Uses the official evaluation script (unchanged)**  
   The script `evaluate_calendar_scheduling.py` (provided by the professor) is treated as a **black box** and is run on each output file to obtain the **real solve rate**.

3. **Ranks models by real performance**  
   - No normalization
   - No rescaling
   - The solve rate is used exactly as reported by the evaluator (e.g. `0.2290`)

4. **Generates an interpretable leaderboard**  
   The leaderboard includes:
   - Rank
   - Model name
   - Parameter size
   - Solve rate
   - Verdict (Best overall / Strong alternative / Acceptable / Not recommended)

5. **Explains *why* a model is better**  
   Strengths and limitations are assigned **based on rank**, not generic text.  
   This provides a clear justification for selecting the top models.

6. **Automatically exports a PDF**  
   A publication-ready file `leaderboard.pdf` is generated using ReportLab.
